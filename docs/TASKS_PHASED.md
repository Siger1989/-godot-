# TASKS_PHASED｜阶段任务卡

规则：Agent 每次只执行当前阶段，不得跳阶段。每阶段结束必须更新 `docs/PROGRESS.md`，并按 `docs/ACCEPTANCE_CHECKLIST.md` 自检。

---

## Phase 0：清洁启动与项目结构

目标：建立干净的模块化工程骨架，不继承旧错误实现。

只允许做：

- 建立目录结构。
- 建立空场景。
- 建立基础脚本文件。
- 建立四房间数据文件。
- 建立 Debug 开关骨架。

禁止：

- 不做可见性三态。
- 不做门洞切线。
- 不做前景遮挡视觉效果。
- 不做灯光高级逻辑。
- 不复制旧 `VisibilityBlendTest.gd` / `VisibilityBlendSection.gd` 的错误逻辑。

输出：

- 推荐目录存在。
- 基础脚本文件存在。
- 数据文件存在。
- 无旧黑片 / 灰片 / 透明片逻辑。

## Phase 1：四房间闭环基础场景

目标：搭出 Room_A、Room_B、Room_C、Room_D，形成一个可以绕圈走的闭环。

只允许做：

- 四个房间。
- 墙、地板、天花。
- 四个 Portal。
- 玩家出生点。
- 目标点、怪物点、道具点、事件点的占位节点。

禁止：

- 不做怪物 AI。
- 不做复杂事件。
- 不做可见性三态。
- 不用房间特例写连接逻辑。

要求：

- Room_A：出生房间，标准风格基准。
- Room_B：灯光测试房间。
- Room_C：目标房间。
- Room_D：稍窄、压迫感测试房间。

---

## Phase 2：玩家与相机

目标：玩家可移动，相机稳定跟随，基础操作清晰。

只允许做：

- 玩家移动。
- 玩家碰撞体。
- 相机跟随。
- 基础输入。

要求：

- 相机斜俯视，建议 45°~55°。
- 玩家位于画面中下或中心偏下。
- 不要太高，不要太远，不要纯上帝视角。

---

## Phase 3：前景遮挡

目标：当前景墙挡住玩家时，墙体挖空或隐藏，玩家始终清晰可见。

只允许做：

- Camera -> Player raycast。
- 命中前景墙 Mesh 隐藏或局部 cutout。
- 保留碰撞体，不影响导航和阻挡。

禁止：

- 不准黄色透明玻璃墙。
- 不准灰色透明大片。
- 不准黑色大三角。
- 不准大范围 alpha 淡出。
- 不准和可见性三态混在一起。

MVP 简化可接受方案：

- raycast 命中的前景墙 Mesh `visible=false`。
- 离开遮挡后恢复 `visible=true`。
- 只隐藏 Mesh，不隐藏 StaticBody / CollisionShape。

---

## Phase 4：材质、灯板、真实灯光、VOID 外墙

目标：后室基础视觉成立。

只允许做：

- 墙纸材质。
- 地板 / 地毯材质。
- 门材质。
- 灯板 Mesh。
- Light3D。
- 外墙 / VOID 黑面。

要求：

- 墙面：黄米色、轻微纹理、不高反光。
- 地面：黄棕或暗米色，纹理清楚不糊。
- 灯板：暖白或白色，简单矩形。
- 真实灯光：偏暖白，略压抑。
- 外墙外侧：黑色，不显示黄色内墙材质。

禁止：

- 灯板因为遮挡直接 `visible=false`。
- 灯板不可见就关闭真实 Light。

---

## Phase 5：门交互与 Portal 状态

目标：门能开关，Portal 能读取门状态。

只允许做：

- `DoorComponent`。
- `PortalComponent`。
- 简单交互提示。
- 门的简单旋转动画。

要求：

- 门状态不写死在房间脚本里。
- Portal 只记录两个区域连接关系和开合状态来源。
- Door 只管门的开关、动画、碰撞。

---

## Phase 6：可见性三态

目标：当前可见 / 已见记忆 / 未知黑区。

只允许做：

- `VisibilitySolver`。
- `VisibilityMemory`。
- `VisibilityRenderer`。

要求：

- 当前可见：正常显示。
- 已见但当前不可见：灰色记忆。
- 从未看过：黑色未知。
- 地板、墙、门、灯板规则一致。

禁止：

- 不准用黑片/灰片盖场景作为正式效果。
- 不准房间 visited 代表整个房间已见。
- 不准 `if Room_A` 特例。

---

## Phase 7：门洞切线可见性

目标：通过门洞看到另一空间的一部分，边界是连续直线切线。

要求：

- 玩家眼点 -> 门洞左边缘。
- 玩家眼点 -> 门洞右边缘。
- 可见区域通过 portal 传播。
- 门关闭时不可透视。
- 门打开时按开口范围通视。

禁止：

- 不准靠近门洞后整房间突然显示。
- 不准用地砖 tile 中心点判断可见性。
- 不准出现马赛克边界。
- 不准透墙。

---

## Phase 8：Debug 与扩展准备

目标：为大场景扩展准备调试工具和数据化结构。

必须有 Debug 开关：

- `show_area_ids`
- `show_portals`
- `show_visible_polygon`
- `show_memory`
- `show_foreground_hits`
- `show_light_ranges`
- `show_void_walls`

要求：

- 每个系统可单独开关。
- 可见性、前景遮挡、灯光、门状态能独立排查。
- 新增房间时只新增 area / portal / prefab / data，不改核心逻辑。
